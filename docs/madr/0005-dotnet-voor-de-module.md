# ADR: `.NET` voor de module

## Status

Accepted

## Context

De technologieën waarmee we willen werken, zoals `RabbitMQ`, kunnen zowel in Java als met `.NET` in C# gebruikt worden.

## Besluit

We zullen `.NET` gebruiken.

Technologisch gezien maakt de keuze weinig verschil. Binnen het team is echter meer bekwaamheid aanwezig in dit gebied, waardoor `.NET` verkozen wordt boven een Java-uitwerking.

## Gevolgen

### Positief

- Met `.NET` is een strakke integratie met Azure en Windows mogelijk.

### Negatief

- Er is een kleinere community dan bij Java.
- Als de teamsamenstelling verandert, moeten nieuwe teamleden kunnen werken met `.NET`.
