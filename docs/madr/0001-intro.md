# ADR: Introductie

## Status

Accepted

## Context

Belangrijke architectuurkeuzes voor de Notification Module worden vastgelegd als **Architecture Decision Records (ADR)** in deze map.

## Besluit

We gebruiken het [Michael Nygard-sjabloon](ADR-Sjabloon.md): Status, Context, Besluit, Gevolgen.

Nieuwe ADR's krijgen een oplopend nummer (`0001`, `0002`, …). Het overzicht staat in [README.md](README.md).

## Gevolgen

### Positief

- Teamleden en beoordelaars kunnen snel zien **waarom** een keuze is gemaakt.
- Superseded ADR's (bijv. [0008](0008-delivery-acknowledgements.md)) blijven bewaard; de opvolger staat vermeld.

### Negatief

- ADR's moeten bij grote wijzigingen worden bijgewerkt of vervangen.
